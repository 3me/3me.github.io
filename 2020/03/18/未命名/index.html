Virtualbox安装注意事项

概要

​ 这篇文章记录自己安装虚拟机过程中的一些心得，以做备忘，避免以后在使用过程中犯同样的错误，同时希望能够对读者有所帮助。

​ 主要介绍下面的内容：

* 虚拟机的选择
* virtualbox网络模式的选择和设置，满足我们host和virtual主机互联互通，同时虚拟机能够连接外网的需求。
* virtualbox增强模式的安装

虚拟机的选择

vmware or virtualbox

​ 一定不要用vmware， 它主要有下面的缺点：

  • 吃内存：以我实际的使用体验，8 G内存电脑开vmware 的同时多开几个浏览器页面，基本上电脑就已经点不动了。
  • 不稳定：vmware经常各种莫名奇妙的问题，比如软件按键点击过于频繁的话，会出现概率性crash。同时对突然断电等问题， 会锁死虚拟文件系统，也会概率性的引起虚拟机不能启动的问题。虽然这些问题是概率性的，但当你着急使用的时候它却罢工，确实是意见让人很崩溃的事情。

​ 相比较而言, virtualbox 开源，小巧，稳定， 内存占用少，后面的网络配置也简单便捷。所以选择virtualbox.

镜像的选择

​ 如果使用ubuntu的话，对于开发者而言，一定要选择服务器版本. ubuntu 桌面版这些年在用户易用性方面做了很多工作，有很明显向 linux 新手系统进化的趋势。但对开发越来越不友好，一些常用的软件命令如 vim, ssh server , ifconfig 等都不会默认安装。因此推荐一步到位，直接安装 ubuntu 服务器版本。

virtualbox网络模式的选择

​ 和大多数虚拟机一样，virtualbox 支持常用的 nat, host, 桥接等网络连接方式。一般选择桥接模式即可满足我们虚拟机和host主机互连，虚拟访问外网的需求。

​ 桥接模式下，VMware虚拟出来的操作系统就像是局域网中的一独立的主机，它可以访问该类网段内任何一台机器。

桥接环境设置需要注意的是 : 在桥接的模式下虚拟机必须与物理机，虚拟网卡需要处于同一网段，(举个例子,物理机IP:192.168.1.2，虚拟机IP:192.168.1.3, 虚拟网卡IP:192.168.1.5)这样虚拟系统才能和真实主机进行通信。
